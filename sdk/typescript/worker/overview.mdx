Alongside the nonce/revocation/rate-limit/account stores, `@afauthhq/worker` ships **`KvAttestedFreshnessStore`** — the KV-backed `AttestedFreshnessStore` for §10.7 [attested sessions](/guides/keep-attested-access-live). Pass it via `attestedSession: { store }`; its KV entries' TTL is set to the remaining window, so lapsed sessions self-evict.

It also ships **`D1SubHUniquenessStore`** — the durable, atomic backend for §10.4.4 [per-principal uniqueness](/concepts/attestation) ("same human, same bucket"). Its composite `(iss, sub_h)` PRIMARY KEY makes `claim()` atomic across isolates (which a KV get-then-put can't do, so there is no KV variant). Apply migration `0002_subh_uniqueness.sql` and pass it via `subHUniqueness`. `createWorker` mirrors `new Server`, so — unlike `defineService` — it does not default this; wire it explicitly to get the guarantee.
