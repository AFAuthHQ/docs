Every reserved code, its HTTP status, the spec section that triggers it, and a one-line note on the client-side response. For the envelope shape itself, see [Error envelope](/concepts/error-envelope).

## All reserved codes (¬ß11.3)

| Code | Status | Triggered when | Spec | Client should |
|---|---|---|---|---|
| `invalid_signature` | `401` | Signature parsing fails, key resolution fails, Ed25519 verify fails, or content-digest doesn't match the body | ¬ß5.5 | Stop ‚Äî the signer is broken |
| `expired_signature` | `401` | `now > expires + skew_tolerance`. Clock drift or you held the signed request too long | ¬ß5.6 | Re-sign and retry |
| `replayed_nonce` | `401` | `(keyid, nonce)` has been seen before within the freshness window | ¬ß5.6 | Stop ‚Äî your signer is reusing nonces (bug) |
| `revoked_key` | `401` | Account DID is on the service's revocation list | ¬ß8.3 | Onboard a fresh agent ‚Äî the old key is gone |
| `invalid_attestation` | `401` | Attestation JWT signature/claim violation, or `iss` not accepted by service | ¬ß10 | Re-mint with a valid attestor |
| `attestation_required` | `401` | Service declares `unclaimed_mode = attested_only` and request lacks/has invalid attestation | ¬ß6.3, ¬ß9.2 | Mint an attestation from an accepted attestor |
| `malformed_request` | `400` | Invalid JSON, missing required fields, invalid DID syntax | ¬ß11.2 | Fix the request shape |
| `unsupported_recipient_type` | `400` | Invitation `recipient.type` isn't in the service's declared `recipient_types` | ¬ß7.2, ¬ß7.7 | Use a type from the discovery doc |
| `unknown_account` | `404` | Account does not exist, and implicit signup is disabled | ¬ß6.4 | Use `POST /accounts` to explicitly sign up |
| `invitation_not_found` | `404`<sup>‚ÄÝ</sup> | Claim token doesn't match a pending invitation | ¬ß7.4 | Stop ‚Äî invitation is gone |
| `invitation_expired` | `410` | Invitation TTL exceeded, or atomically replaced by a newer one | ¬ß7.3 | Send a fresh invitation |
| `account_expired` | `410` | Account exceeded `unclaimed_ttl_seconds` without being claimed | ¬ß6.1 | Onboard a fresh agent |
| `already_claimed` | `409` | Operation requires `UNCLAIMED` / `INVITED` state but account is already `CLAIMED` | ¬ß11.2 | Stop ‚Äî claim has already happened |
| `not_claimed` | `409` | Operation requires `CLAIMED` state but account isn't claimed yet | ¬ß11.2 | Send an owner invitation first |
| `owner_authentication_required` | `403` | Operation requires an owner-authenticated session; none was supplied (or supplied identity doesn't match the match relation) | ¬ß7.4 | Prompt the human to authenticate at the claim page |
| `owner_session_too_stale` | `403` | Owner session present but `authenticatedAt` is missing or older than the ¬ß7.5 freshness window (60-300s) | ¬ß7.5 | Prompt the human to re-authenticate, then retry |
| `owner_binding_blocked` | `403` | An agent-signed request attempted an owner-binding operation post-claim | ¬ß7.5 | Stop ‚Äî this operation requires the owner |
| `rate_limit_exceeded` | `429` | Per-route per-DID rate-limit bucket exhausted | ¬ß11.3 | Honour the `Retry-After` header |

<sup>‚ÄÝ</sup> Status for `invitation_not_found` is not explicitly bound in spec text. `404` is the conventional reading; a service may also use `410` if the invitation was previously valid and atomically superseded. Don't program against the status alone ‚Äî inspect `code`.

## Grouped by what you should do about it

### Transient ‚Äî retry with a small change

- `expired_signature` ‚Äî re-sign, retry.
- `rate_limit_exceeded` ‚Äî sleep `Retry-After` seconds, retry.
- `owner_session_too_stale` ‚Äî re-authenticate the human, retry.

### Misconfigured ‚Äî surface to the developer

- `replayed_nonce` ‚Äî your signer is buggy.
- `malformed_request` ‚Äî fix the request shape.
- `unsupported_recipient_type` ‚Äî read the discovery doc.
- `invalid_attestation` / `attestation_required` ‚Äî fix attestor wiring.

### Onboarding required

- `unknown_account` (implicit signup disabled) ‚Äî call `POST /accounts`.
- `revoked_key` / `account_expired` ‚Äî start over with a fresh agent.

### State machine boundary

- `already_claimed` / `not_claimed` ‚Äî read the spec state machine; the operation doesn't apply.
- `owner_authentication_required` ‚Äî needs the human in the loop.
- `owner_binding_blocked` ‚Äî needs the human in the loop, full stop.

### Auth surface

- `invalid_signature` ‚Äî programmer error or hostile input.
- `invitation_expired` / `invitation_not_found` ‚Äî invitation is gone; re-issue.

## Custom codes

Services MAY add their own codes beyond ¬ß11.3 (`example_quota_exceeded`, `feature_not_in_plan`, etc.) and SHOULD prefix them with a service-specific namespace. The envelope shape is the same ‚Äî clients can ignore unknown codes safely.
